## Layer Descriptions

| Layer | Function | Responsibility |
|-------|----------|---------------|
| **Data** | `WeatherData` | Immutable data container with typed fields |
| **API** | `fetch_weather()` | HTTP communication, JSON parsing, error handling |
| **Business** | `check_alert()` | Threshold-based alert classification |
| **Domain** | `get_rainfall_category()` | CMA classification (hydrology knowledge) |
| **Persistence** | `log_alert()` | Timestamped file logging with context manager |
| **Orchestration** | `main()` | Iterates cities, calls layers, formats output |

## Data Flow

```
OpenWeatherMap API
    ↓ (HTTP GET with params)
fetch_weather() → requests.get() → response.json()
    ↓
WeatherData dataclass (city, temperature, humidity, rainfall, description, timestamp)
    ↓
check_alert(rainfall) → alert level + status
    ↓
get_rainfall_category(rainfall) → CMA string
    ↓
log_alert() [if Red] → outputs/alert_log.txt
    ↓
main() → formatted console output
```

## Migration Changes from Legacy v1

```
v1: urllib2.urlopen(string_url)     → v2: requests.get(url, params=dict, timeout=int)
v1: dict access without checks       → v2: .get(key, default) safe access
v1: raw tuple return                 → v2: @dataclass typed return
v1: open/close without context       → v2: with open() as f (auto-close)
v1: undefined variable scope         → v2: explicit function parameters
v1: no error handling                → v2: try/except + raise_for_status()
```
