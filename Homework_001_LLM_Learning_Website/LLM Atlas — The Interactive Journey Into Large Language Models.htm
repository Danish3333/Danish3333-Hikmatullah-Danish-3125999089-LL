🗺️ What is an LLM?

A Large Language Model is a function that takes a sequence of tokens (numbers) and outputs a probability distribution over the next token. Train it on enough text, and it becomes the technology behind ChatGPT, Claude, and Gemini.

🌱 Beginner Explanation

Think of an LLM like a super-powered autocomplete. When you type "The cat sat on the...", your phone suggests "mat" because it has seen that phrase many times. An LLM does the same thing — but with the entire internet as training data, and billions of parameters to capture patterns.

The key insight: all the complexity is just doing this autocomplete very, very well. Every step in the pipeline is in service of answering one question: "Given everything I've seen so far, what token is most likely next?"

🔤

Tokenization

Converts raw text into integer IDs via BPE (Byte Pair Encoding).

"unbelievable" → ["un", "believ", "able"]
🧠

Self-Attention

Each token computes Q, K, V vectors and attends to all others.

Attn = softmax(QKᵀ / √d_k) · V
⚡

Feed-Forward

A 2-layer MLP applied independently per token. Stores most "knowledge".

FFN(x) = SwiGLU(xW₁) ⊙ (xW₃) · W₂

How BPE Works

1
Start with characters: every character is its own token. l o w e r
2
Count adjacent pairs: "l o", "o w", "w e", "e r" → count frequency in corpus
3
Merge most frequent: if "e r" wins → merge to "er": l o w er
4
Repeat until vocab size: typically 32K–128K tokens
BPE merges for "lower": Step 1: l o w e r → l o w er (merged "e"+"r") Step 2: l o w er → l ow er (merged "o"+"w") Step 3: l ow er → low er (merged "l"+"ow") Step 4: low er → lower (merged "low"+"er") Result: ["lower"] → token_id: 2847

🐱 Cat & Mouse: The Famous Worked Example

This is the example every LLM teacher loves. We compute attention scores for the sentence "The cat chases the mouse" by hand — with explicit numbers at every step.

Multi-Head Attention

Run h attention heads in parallel. Each head learns a different "view" of the relationships.

MultiHead(Q,K,V) = Concat(head₁,...,head_h) · W^O where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V) GPT-2 small: d_model=768, h=12, d_k=64

Causal (Masked) Attention

Decoder-only models mask future tokens. Token i can only attend to positions ≤ i.

mask[i][j] = -∞ if j > i mask[i][j] = 0 if j ≤ i → softmax(-∞) = 0, future tokens get zero weight

Cross-Attention

Encoder-decoder models (T5, BART) have Q from decoder, K & V from encoder.

CrossAttn(Q_dec, K_enc, V_enc) = softmax(Q_dec · K_encᵀ / √d_k) · V_enc → Decoder "reads" the encoded input

📈 Positional Encoding — Order Matters

Transformers have no inherent sense of word order. "Dog bites man" and "Man bites dog" have identical token sets. Positional encodings fix this.

PE(pos, 2i) = sin(pos / 10000^(2i/d)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

Hover any cell to see the exact value.

Sinusoidal (Original)

PE(pos, 2i) = sin(pos/10000^(2i/d)) PE(pos, 2i+1) = cos(pos/10000^(2i/d))

Fixed. No parameters. Poor length extrapolation.

RoPE ⭐

RoPE: rotates Q and K by position-dependent angles → attention(q_m, k_n) depends only on (m−n) θ_j = 1 / 10000^(2j/d)

Used by LLaMA, Mistral, GPT-NeoX. Good extrapolation.

ALiBi

attn(i,j) = q_i · k_j − m · |i − j| m = head-specific slope (geometric) No learned position params

Used by BLOOM, MPT. Excellent extrapolation.

⚡ Feed-Forward Network — Where Knowledge Lives

After attention mixes information across tokens, the FFN processes each token independently. It contains ~2/3 of all parameters — this is where "facts" are stored.

Standard ReLU FFN

FFN(x) = max(0, x·W₁ + b₁) · W₂ + b₂ W₁ ∈ ℝ^(d_model × d_ff), W₂ ∈ ℝ^(d_ff × d_model) Typically d_ff = 4 × d_model

GLU (Gated Linear Unit)

GLU(x) = (x·W₁) ⊙ σ(x·W_gate) FFN_GLU(x) = GLU(x) · W₂ A learnable gate controls information flow.

SwiGLU — The Modern Default

SwiGLU(x) = SiLU(x·W₁) ⊙ (x·W₃) FFN_SwiGLU(x) = SwiGLU(x) · W₂ SiLU(x) = x · σ(x)

Used in LLaMA, PaLM, Mistral. +0.5–1% over ReLU at same param budget.

Flash Attention — The IO Revolution

Standard attention is O(n²) memory because it materializes the full score matrix. Flash Attention computes attention in tiles using fast SRAM, achieving O(n) memory with identical math.

Standard: O(n²d) FLOPs, O(n²) memory — bottlenecked by HBM bandwidth FlashAttn: O(n²d) FLOPs, O(n) memory — tiles in fast SRAM FlashAttn-2 (2023): 2× over FA-1 FlashAttn-3 (2024): WGMMA async, targets H100

🌳 Mixture of Experts — Sparse Routing

MoE scales parameters without scaling FLOPs. Each token is routed to only a few "expert" FFN layers via a learned router. Click "Route next token" to see the gating in action.

🌳

Interactive MoE Router

8 experts, top-2 routing (only 2 fire per token). Watch different tokens route to different experts based on their meaning.

Routing Math

Router: g(x) = softmax(W_g · x) ∈ ℝ^E Top-k selection: keep top-k g(x), zero out rest Output: y = Σ_{i ∈ top-k} g_i(x) · E_i(x) E = number of experts (e.g., 8, 64, 256) k = experts activated per token (typically 2) Load balancing loss: L_balance = α · E · Σ_{i=1}^E f_i · p_i f_i = fraction of tokens routed to expert i p_i = avg router probability for expert i

Mixtral 8×7B: 47B total params but only ~13B active per token.

🏋️ Training Pipeline — From Raw Data to Aligned Model

Stage 1: Pre-training

L = −1/T · Σₜ log P(xₜ | x₁, ..., xₜ₋₁; θ) GPT-3: 175B params · 300B tokens · ~3.14×10²³ FLOPs Chinchilla: optimal D ≈ 20 × N

Next-token prediction across trillions of tokens. Uses AdamW, mixed precision (bf16), tensor + pipeline + data parallelism across thousands of GPUs.

Stage 2: Supervised Fine-Tuning (SFT)

L_SFT = −Σₜ∈response log P(xₜ | context; θ) Typically 10K–100K high-quality (instruction, response) pairs.

Curated examples teach the model to follow instructions. Loss is computed only on response tokens.

Stage 3: Alignment (RLHF / DPO)

See the RLHF simulator for an interactive walk-through.

SFT loss: cross-entropy on response tokens RLHF loss: L = E[r(y)] − β·KL(π_θ ∥ π_ref) DPO loss: L = −log σ(β·log(π_θ(y_w)/π_ref(y_w)) − β·log(π_θ(y_l)/π_ref(y_l)))

Scaling Laws (Chinchilla)

L(N, D) = E + A/N^α + B/D^β α ≈ β ≈ 0.5 Compute-optimal: D* ≈ 20 × N

LLaMA showed that smaller models on MORE tokens (7B on 1T) can beat larger compute-optimal models, because inference cost matters at deployment.

🎯 RLHF — Interactive Simulator

Step 1: Collect Human Preferences

You are the human. Pick which response is better. This trains the reward model.

Step 2: Reward Model Training

The reward model learns to predict human preferences using the Bradley-Terry model.

P(A ≻ B) = σ(r(A) − r(B)) Reward Model Loss: L = −E[log σ(r(y_w) − r(y_l))]

Step 3: Policy Optimization

L = E[r(y)] − β · KL(π_θ ∥ π_ref) PPO clipped surrogate objective prevents large updates. The KL penalty ensures we don't drift too far from SFT.
L_DPO = −log σ(β·log(π_θ(y_w)/π_ref(y_w)) − β·log(π_θ(y_l)/π_ref(y_l))) No reward model needed! Direct preference optimization (Rafailov et al., 2023).
